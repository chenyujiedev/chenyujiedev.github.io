二、第 2 周：图像处理概述

1.5 图像的形成过程

1）核心概念

  • 图像可以理解为：由许多同时具有位置值和亮度值的像素组成的集合。
  • 图像形成时，光源发光，物体反射光，成像系统接收反射光，最终形成图像。课件给出的公式是：

其中：

  • f(x, y)：图像在 (x, y) 位置的亮度值
  • i(x, y)：照明强度
  • r(x, y)：物体反射系数。

2）采样与量化

采样（sampling）

  • 把原本连续的空间位置离散化。
  • 采样决定图像有多少个像素，也就是图像的空间分辨率。

量化（quantization）

  • 把连续的亮度值离散化，用有限个数值来表示亮度。
  • 量化决定每个像素最终能取哪些灰度值。

3）辅助理解

  • 数字图像通常表示成 M × N 大小。
  • 若亮度使用 k bit 量化，则共有 2^k 个亮度级。
  • 8 bit 图像有 2^8 = 256 个亮度级。

4）必会问答

什么是图像形成公式？

图像亮度 f(x, y) 由照明强度 i(x, y) 和物体反射系数 r(x, y) 的乘积决定，即 f(x, y) = i(x, y) × r(x, y)。

采样和量化的区别是什么？

  • 采样处理的是位置，决定像素个数。
  • 量化处理的是亮度，决定像素取值。

数字图像为什么是离散的？

因为连续的空间位置经过采样，连续的亮度值经过量化，最后变成有限个像素和有限个灰度级。

5）一句话记忆

采样决定像素个数，量化决定像素取值。

三、第 3 周：Python 基础

3.3 运算符

1）基本概念

  • 表达式（expression）：能产生结果或值的代码片段。
  • 运算符（operator）：执行运算的符号。
  • 操作数（operand）：参与运算的对象。

2）运算符分类

算术运算符

赋值运算符

比较运算符

逻辑运算符

位运算符

成员运算符

身份运算符

is, is not。

3）优先级要点

  • ** 优先级高。
  • 单目运算优先于普通加减。
  • 乘除类一般高于加减类。
  • 比较运算在算术运算之后。
  • 逻辑运算中通常 not > and > or。

4）切片（slice）运算符

特点：

  • 结束索引不包含在结果中
  • 默认步长是 1
  • 步长为负数表示反向切片。

5）代码例子：切片运算符

常见切片例子

a[:2]：取前 2 个元素

a[4:-1]：从索引 4 取到倒数第 2 个

a[2::2]：从索引 2 开始，每隔 2 个取 1 个

a[::-1]：整个列表倒序

a[:1:-1]：倒着取，直到索引 1 之前停止

a[7:1:-2]：从 7 到 1，步长为 -2

a[:-4:-1]：从后往前取到倒数第 4 个之前

6）必会问答

什么是表达式、运算符、操作数？

表达式是能产生值的代码；运算符是执行运算的符号；操作数是参与运算的对象。

切片最容易错在哪里？

最容易错在“结束索引不取”，也就是左闭右开。

7）一句话记忆

切片左闭右开，结束不取，步长可正可负。

3.4 基本语句

1）条件语句

  • if 后面写条件。

  • 条件后面必须有冒号 :

  • elif 可以有多个。

  • else 可以省略。

  • 条件语句常和比较运算符、逻辑运算符一起使用。

  • 缩进必须正确，因为 Python 用缩进区分代码块。

% 是取余运算

能被 4 整除且不能被 100 整除：闰年

能被 400 整除：也是闰年

if → elif → else 是按顺序判断

2）循环语句

while

  • 条件为真时反复执行。

for

  • 更适合对一组数据逐个遍历。

while：条件为真就继续循环

break：中途直接跳出循环

n = n - 1：控制循环次数

含义：最多输入 4 次，如果中间输入 0 就提前结束

3）遍历

for ~ in 用于遍历序列对象中的元素，可遍历：

  • 列表
  • 元组
  • 字符串
  • 字典等。

常见搭配：

  • range()：按区间/索引遍历

  • enumerate()：同时得到索引和值

  • zip()：把多个序列配对后一起遍历。

max()：求最大值

min()：求最小值

abs()：求绝对值

pow(a, b)：求幂

divmod(a, b)：同时得到商和余数

map()：把同一种操作依次作用到一组数据上

4）PPT 例子要会看

  • 闰年判断：看 if / elif / else 的执行顺序。
  • while 例子：看“条件成立就持续重复”。
  • for + range / enumerate / zip：看三种不同遍历方式分别在做什么。

5）必会问答

Python 为什么不靠大括号区分代码块？

因为 Python 用缩进表示代码块层级，缩进本身就是语法结构。

if / elif / else 的执行顺序是什么？

从上到下依次判断；一旦某个条件成立，就执行对应代码块，后面的分支不再执行。

为什么缩进错误会导致程序错误？

因为缩进在 Python 中不是排版问题，而是语法结构本身；缩进错了，代码块结构就错了。

6）一句话记忆

if 看条件，while 看真假反复执行，for 看逐个取元素。

3.5 函数和库

1）函数（function）

  • 函数是为完成某个功能而写的一组可重复使用的语句。

  • 使用函数可以减少重复代码，提高复用性。

  • def 用于定义函数。

  • 参数是输入，return 是返回值。

  • 一个函数可以返回多个值。

def：定义函数

calc_area(...)：带参数函数

c=None：默认参数

return result, msg：返回多个值

type=1, a=5, b=5：关键字参数调用

area, msg = ...：接收多个返回值

_：表示这个返回值不重点使用

2）模块（module）与包（package）

  • 模块：一个 Python 文件，里面可以定义变量、函数、类。

  • 包：多个模块组成的集合。

  • import 用于导入模块或包。

import math：导入整个模块

import math as m：导入并起别名

from math import pi, sqrt：只导入模块中的部分内容

3）内置函数（built-in function）

常见内置函数：

max()：求最大值

min()：求最小值

abs()：求绝对值

pow(a, b)：求幂

divmod(a, b)：同时得到商和余数

map()：把同一种操作依次作用到一组数据上

4）PPT 例子要会看

  • 函数定义例子：看 def、参数、返回值、默认参数。
  • 模块导入例子：看 import ... as ...、from ... import ...
  • 内置函数例子：看这些函数的作用，而不是死记代码。

5）必会问答

函数、模块、包之间的关系是什么？

函数是完成具体功能的代码块；模块是装这些函数的文件；包是多个模块组成的集合。

为什么程序要模块化？

为了减少重复代码，便于组织、复用和维护。

6）一句话记忆

函数是功能块，模块是文件，包是模块集合。

四、第 4 周：OpenCV 接口 Part 1

4.1 窗口控制（window control）

1）为什么需要窗口控制

图像处理本质上是对二维矩阵做运算。只看矩阵元素，不容易直观看出结果，所以需要把矩阵显示成窗口中的图像。OpenCV 中只有窗口处于激活状态时，鼠标和键盘事件才能被检测到。

2）常见函数作用

  • cv2.namedWindow()：创建并命名窗口
  • cv2.imshow()：在窗口中显示图像
  • cv2.waitKey()：等待键盘输入，也让窗口保持显示
  • cv2.destroyWindow()：关闭指定窗口
  • cv2.destroyAllWindows()：关闭全部窗口
  • cv2.moveWindow()：移动窗口位置
  • cv2.resizeWindow()：调整窗口大小。

3）WINDOW_AUTOSIZE 与 WINDOW_NORMAL

WINDOW_AUTOSIZE

  • 窗口大小与图像大小绑定
  • 不可随意调整窗口大小。

WINDOW_NORMAL

  • 窗口大小可调整。

关键点

  • 窗口大小变化，不代表原始图像矩阵被改变。

4）PPT 例子

例子 1：窗口移动

  • 先生成一个 0 元素矩阵图像，再用切片把像素设为某一亮度。
  • 创建两个窗口，并移动到不同坐标位置。
  • 这个例子说明：窗口有名字、有位置，位置可控制。

例子 2：AUTOSIZE vs NORMAL

  • 用两个窗口显示同一图像。
  • AUTOSIZE 不能自由改窗口尺寸。
  • NORMAL 可以调整尺寸。
  • 但图像矩阵本身没有变。

5）必会问答

为什么图像处理需要窗口？

因为图像处理是在改矩阵，只看矩阵不直观，通过窗口显示后才能直接观察处理效果。

WINDOW_AUTOSIZE 和 WINDOW_NORMAL 的区别是什么？

前者窗口尺寸跟图像绑定，后者窗口可调整；但窗口变化不等于图像矩阵变化。

6）一句话记忆

窗口是用来把矩阵结果可视化；AUTOSIZE 绑图像，NORMAL 可改大小。

4.2 事件处理函数

1）什么是事件

事件是程序能够检测并处理的动作或情况，例如：

  • 键盘按键
  • 鼠标移动
  • 鼠标点击
  • trackbar 滑动条变化
  • 更广义上还可以包括定时器等事件。

2）什么是回调函数（callback）

回调函数不是开发者在主流程里直接调用，而是当事件发生时，由系统自动调用事先注册的函数。

3）OpenCV 中常见事件类型

  • 键盘事件
  • 鼠标事件
  • trackbar 事件。

4）键盘事件

cv2.waitKey(delay)

作用：

  • 等待指定时间内的键盘输入
  • 处理键盘事件
  • 防止窗口一闪而过。

要点：

  • delay <= 0：一直等待，直到有按键输入
  • delay > 0：等待指定时间，没有按键也会继续执行。

关键细节：

  • 想触发键盘事件，必须先点击窗口让其激活，再按键。

5）鼠标事件

cv2.setMouseCallback()

  • 用来注册鼠标事件回调函数。

PPT 例子说明：

  • 鼠标点击位置会被记录。
  • 可以根据点击位置在图像上进行操作。
  • 深化例题中，使用两次右键点击确定两个点，再计算圆心和半径，并调用 cv2.circle() 画圆。

6）Trackbar 事件

cv2.createTrackbar()

  • 用来创建滑动条并注册回调函数。

PPT 例子说明：

  • 通过滑动条位置改变亮度值。
  • 滑动条变化时，回调函数被触发。
  • 回调里再修改整幅图像的像素值，因此图像亮度随滑动条变化。

7）必会问答

什么是回调函数？

回调函数是事件发生时由系统自动调用的函数，而不是程序员在主流程中直接调用。

cv2.waitKey() 的作用是什么？

接收键盘输入、处理键盘事件，并让窗口保持显示，不会立刻关闭。

Trackbar 事件本质上在做什么？

滑动条位置改变时触发回调函数，再由回调函数修改图像或参数。

8）一句话记忆

事件是动作，回调是事件发生时系统自动调用的函数。

五、第 5 周：OpenCV 接口 Part 2

4.4 图像文件处理（image file processing）

1）图像文件处理流程

图像文件处理的基本流程：

  1. 从图像文件中读入矩阵
  2. 在矩阵上做处理
  3. 必要时直接查看矩阵元素
  4. 将处理后的结果再保存成图像文件。

2）两个最核心的函数

cv2.imread(filename, flags)

  • 读取图像文件并转换成矩阵。

cv2.imwrite(filename, img, params)

  • 把矩阵保存为图像文件。

3）imread() 常见 flag

  • cv2.IMREAD_UNCHANGED：按原类型读取，包含 alpha 通道
  • cv2.IMREAD_GRAYSCALE：按灰度图读取
  • cv2.IMREAD_COLOR：按彩色图读取
  • cv2.IMREAD_ANYDEPTH：按图像原有深度读取
  • cv2.IMREAD_ANYCOLOR：按图像原有颜色类型读取。

4）读取后要会观察什么

读图后要关注：

  • 图像是否读取成功
  • depth
  • channel
  • mat_type
  • 某个像素位置的值。

5）PPT 例子要会看

例子 1：同一张图按灰度图和彩色图读取

  • 灰度读取后一般是 1 通道
  • 彩色读取后一般是 3 通道
  • 同一像素位置，灰度值是一个数，彩色值一般是三个通道值。

例子 2：用辅助函数打印图像信息

PPT 中通过自定义的 print_matInfo() 打印：

  • depth
  • channels
  • mat_type。
    这个例子说明：图像文件处理不只是“看图”，还要会观察其矩阵属性。

例子 3：模块导入

课件中把辅助函数放进 Common.utils 模块，然后在别的文件中导入使用，说明模块化可以让多个章节共用同一工具函数。

6）保存图像时要注意

  • imwrite() 保存格式由扩展名决定，例如 JPG、BMP、PNG、TIF 等。
  • params 可用于设置压缩、质量等参数。

7）必会问答

cv2.imread() 的作用是什么？

从图像文件中读取图像，并把它转换为矩阵。

cv2.imwrite() 的作用是什么？

把矩阵保存为图像文件。保存格式由扩展名决定，附加参数可用于控制压缩和质量。

彩色图读取与灰度图读取的区别是什么？

彩色图通常得到 3 通道矩阵，灰度图通常得到 1 通道矩阵；同一像素在灰度图里是一个值，在彩色图里通常是三个通道值。

8）一句话记忆

imread 读图成矩阵，imwrite 把矩阵存回图像文件。

六、第 6 周：OpenCV 基本矩阵运算

5.3 算术运算函数

1）核心概念：逐元素运算

5.3 最重要的核心是：

两个矩阵做运算时，是对应位置的元素与对应位置的元素分别运算。

这叫：

  • per-element
  • element-wise 运算。

2）常见四则运算函数

cv2.add(src1, src2[, mask])

  • 对应元素做加法。

cv2.subtract(src1, src2[, mask])

  • 对应元素做减法。

cv2.multiply(src1, src2[, scale])

  • 对应元素做乘法
  • scale 是附加缩放系数。

cv2.divide(src1, src2[, scale])

  • 对应元素做除法。

cv2.addWeighted(src1, alpha, src2, beta, gamma)

  • 先分别乘权重，再做加权和。
  • 常用于两幅图像融合。

3）饱和处理（saturation）

OpenCV 在算术运算里强调 saturation：

  • 结果超过上限时取最大值
  • 结果低于下限时取最小值。

结合相邻章节的亮度变化例子也能帮助理解：

  • 超过 255 的值会被截到 255
  • 低于 0 的值会被截到 0。

4）mask 的作用

mask 的作用是：

只在 mask 非 0 的位置执行运算。

也就是说，mask 可以限制“运算只发生在指定区域”。

5）PPT 例子要会看

例子 1：mask 限定区域做加法

  • 先构造两个矩阵
  • 再构造一个 mask
  • 普通加法会对全部元素加
  • 带 mask 的加法只会对 mask 指定区域进行运算。

例子 2：整型除法与浮点除法

  • 直接对整型矩阵做 divide() 时，小数部分会丢失
  • 若先转成 float32，则能保留小数部分。
    这道题很容易考概念。

例子 3：加权融合

  • addWeighted() 对两幅图分别乘权重，再求和
  • 常用来实现图像融合或叠加效果。

6）位运算（逻辑运算）

课件在 5.3.3 中还讲了逻辑/位运算函数：

  • cv2.bitwise_and()
  • cv2.bitwise_or()
  • cv2.bitwise_xor()
  • cv2.bitwise_not()。

用途：

  • 配合前景 mask、背景 mask，提取指定目标区域
  • 用于图像合成、目标抠取等。

7）必会问答

什么叫逐元素运算？

两个矩阵中对应位置的元素分别参与运算，而不是把整个矩阵当成一个整体去算。

add / subtract / multiply / divide 分别做什么？

它们分别做逐元素的加、减、乘、除运算。

mask 的作用是什么？

mask 用来限制运算区域，只有 mask 非 0 的位置才参与运算。

算术运算和位运算的区别是什么？

  • 算术运算关注数值大小变化，如加减乘除
  • 位运算关注二进制层面的逻辑处理，常配合 mask 做区域提取或合成。

8）一句话记忆

算术运算看数值变化，位运算看逻辑关系，mask 用来限制运算区域。

七、第 7 周：像素处理

6.3 直方图（histogram）

1）什么是直方图

直方图表示的是：

图像中每个像素值出现了多少次。

也就是像素值的频数分布。

2）横轴与纵轴

  • 横轴：灰度级 / 像素值范围
  • 纵轴：频数（出现次数）。

3）为什么直方图重要

直方图能帮助分析：

  • 图像整体偏亮还是偏暗
  • 对比度是否充足
  • 是否需要进一步增强处理。

4）6.3.2 直方图计算

直方图本质上就是：

统计每个像素值出现的次数。

PPT 中既有手写统计方式，也有 OpenCV 函数方式。

5）cv2.calcHist() 的作用与参数

函数原型重点理解：

作用：

  • 计算矩阵元素值的频数，也就是计算直方图。

参数含义：

  • images：输入图像数组
  • channels：参与计算的通道列表
  • mask：只统计指定区域时使用的掩膜
  • histSize：每个维度的直方图大小，也就是 bin 个数
  • ranges：每个维度的数值范围
  • accumulate：累积标志；从多个数组中求一个累计直方图时使用。

6）PPT 例子要会看

例子 1：手写单通道直方图

  • 遍历图像像素
  • 把像素值作为索引
  • 对应 bin 的计数加 1。
    这个例子说明：直方图的本质就是“计数”。

例子 2：用 calcHist() 计算直方图

  • 给定图像、通道、bin 数和范围
  • 得到对应直方图。
    这个例子说明：OpenCV 已经把统计过程封装好了。

例子 3：把直方图画出来

PPT 里有根据直方图数据绘图的例子，说明“直方图不只是一个数组，也可以画成图像或柱状图，用来直观看分布形态”。

例子 4：彩色图像的颜色直方图

PPT 中还有彩色直方图示例，说明不同通道都可以分别统计。

7）直方图与亮暗分布

  • 如果直方图大量集中在高灰度区域，图像通常偏亮。
  • 如果直方图大量集中在低灰度区域，图像通常偏暗。
  • 如果直方图分布很窄，通常说明图像对比度不高。

8）6.3.4 直方图拉伸（histogram stretching）

含义

  • 当直方图分布过窄时，图像对比度差。
  • 通过拉伸把原本较窄的灰度分布扩展到更大的范围，从而增强对比度。

PPT 例子想说明什么

  • 先找出有效灰度范围中的最小值和最大值
  • 再把这一区间映射到更大的灰度范围。

复习时记住

  • 拉伸强调的是：扩大分布范围
  • 适合处理“直方图挤在一起”的低对比度图像。

9）6.3.5 直方图均衡化（histogram equalization）

含义

  • 直方图均衡化的目标是让灰度分布更均匀，从而增强图像的明暗对比。

PPT 中给出的步骤

  1. 计算图像直方图
  2. 根据频数计算累计频数（累计和）
  3. 将累计频数归一化
  4. 用归一化后的累计分布去映射新的像素值。

效果

  • 提高图像可辨识度
  • 改善视觉质量。

10）拉伸与均衡化的区别

直方图拉伸

  • 重点是把分布拉开
  • 更强调扩大灰度范围。

直方图均衡化

  • 重点是把分布重新分配得更均匀
  • 更强调整体分布的再调整。

11）必会问答

什么是直方图？

直方图表示图像中各个像素值出现的次数，也就是像素值的频数分布。

cv2.calcHist() 的作用是什么？

用于计算图像的直方图，也就是统计像素值频数分布。

为什么直方图重要？

因为它可以反映图像是偏亮还是偏暗、对比度是否足够，并为图像增强提供依据。

直方图拉伸和直方图均衡化有什么区别？

  • 拉伸是把原本较窄的灰度范围拉开
  • 均衡化是重新分配灰度分布，使其更均匀。

直方图均衡化的基本步骤是什么？

先算直方图，再算累计频数，再归一化累计分布，最后根据归一化结果映射新的像素值。

12）一句话记忆

直方图看分布；拉伸扩范围；均衡化重分配。

八、老师高频概念题答案

1）定义类

什么是图像形成过程？

图像形成过程是指光源发光、物体反射光、成像系统接收反射光并形成图像的过程，图像亮度满足 f(x, y)=i(x, y)×r(x, y)。

什么是表达式、运算符、操作数？

表达式是能产生值的代码片段；运算符是执行运算的符号；操作数是参与运算的对象。

什么是回调函数？

回调函数是事件发生时由系统自动调用、而不是开发者在主流程中直接调用的函数。

什么是直方图？

直方图是图像像素值频数分布图，表示每个灰度值出现了多少次。

2）区别类

采样 vs 量化

  • 采样决定像素个数
  • 量化决定像素取值。

彩色图读取 vs 灰度图读取

  • 彩色图通常是 3 通道
  • 灰度图通常是 1 通道。

WINDOW_AUTOSIZE vs WINDOW_NORMAL

  • AUTOSIZE：窗口大小受图像约束
  • NORMAL：窗口大小可调整。

算术运算 vs 位运算

  • 算术运算看数值变化
  • 位运算看二进制逻辑处理。

直方图拉伸 vs 直方图均衡化

  • 拉伸：扩大灰度分布范围
  • 均衡化：重新分配灰度分布，使其更均匀。

3）作用类

cv2.imshow() 的作用

在指定窗口中显示图像。

cv2.waitKey() 的作用

等待按键输入、处理键盘事件，并让窗口保持显示。

cv2.imread() 的作用

把图像文件读成矩阵。

cv2.imwrite() 的作用

把矩阵保存为图像文件。

cv2.calcHist() 的作用

计算图像的直方图，即像素值的频数分布。

mask 的作用

限制运算区域，只在指定位置执行运算。
