三、第1章复习提纲：计算机系统概述

3.1 必背核心概念

1）计算机系统的组成

  • 硬件（hardware）
  • 系统软件（system software）
  • 应用软件（application software）

2）计算机的基本功能

  • 按程序顺序执行
  • 读取数据（read）
  • 处理数据（process）
  • 存储数据（store）

3）计算机三大核心部件

  • CPU（中央处理器）
  • 存储器
  • 输入输出设备（I/O）

4）主存与辅存的区别

主存（Main Memory）

  • 靠近CPU
  • 速度快
  • 容量较小
  • 价格高
  • 断电后内容消失

辅存（Auxiliary Storage）

  • 可长期保存
  • 容量大
  • 价格低
  • 速度慢
  • 例如：硬盘、SSD、CD-ROM、闪存

5）信息表示与程序语言

  • 计算机内部所有信息本质上都是 二进制位
  • 程序语言分为：
    • 机器语言
    • 汇编语言
    • 高级语言
  • 翻译程序：
    • 高级语言 → 机器语言：编译器（compiler）
    • 汇编语言 → 机器语言：汇编器（assembler）

6）指令格式基础

一条机器指令一般包含：

  • 操作码（op code）
  • 操作数（operand）

常考理解：

  • 若操作码占 n 位，则最多表示 2^n 种操作
  • 若地址字段占 m 位，则最多可寻址 2^m 个地址单元

7）系统总线

系统总线分为三类：

  • 地址总线（Address Bus）
  • 数据总线（Data Bus）
  • 控制总线（Control Bus）

重点区分：

  • 地址总线：通常单向
  • 数据总线：通常双向
  • 控制总线：通常双向

8）CPU与存储器/IO连接

必须搞清：

  • 存储器读
  • 存储器写
  • I/O读
  • I/O写
  • 设备控制器作用
  • 状态寄存器、数据寄存器

3.2 第1章最容易出题的点

高频点1：操作码和地址位数计算

例如：

  • 操作码 6 位 → 最大操作种类 2^6 = 64
  • 地址字段 11 位 → 可寻址 2^11 = 2048

高频点2：字长、字节换算

必会：

  • 8 bit = 1 byte
  • 32 bit = 4 byte
  • 存储容量换算题

高频点3：总线宽度与寻址能力

例如：

  • 地址总线 16 位 → 可寻址 2^16 = 64K
  • 数据总线 32 位 → 一次可传送 32 bit

高频点4：主存、辅存、I/O区别

很适合出选择题、简答题

高频点5：高级语言 → 汇编 → 机器语言

老师很容易考“翻译关系”和“各自特点”

四、第2章复习提纲：CPU的结构与功能

4.1 必背核心概念

1）CPU三大组成

  • ALU
  • 寄存器组
  • 控制单元（Control Unit）

2）CPU的基本功能

  • 取指（Instruction Fetch）
  • 译码（Instruction Decode）
  • 取数（Data Fetch）
  • 数据处理（Data Process）
  • 结果存储（Data Store）

3）指令周期（Instruction Cycle）

一条指令的完整执行过程叫指令周期，通常包括：

  • 取指周期（Fetch Cycle）
  • 执行周期（Execution Cycle）

有时还包括：

  • 中断周期（Interrupt Cycle）
  • 间址周期（Indirect Cycle）

4.2 必背寄存器

核心寄存器

  • PC（Program Counter）
    • 存下一条将要取出的指令地址
  • IR（Instruction Register）
    • 存当前取出的指令
  • AC（Accumulator）
    • 暂存数据和运算结果
  • MAR（Memory Address Register）
    • 存待访问的存储器地址
  • MBR（Memory Buffer Register）
    • 存读出或写入的数据
  • SP（Stack Pointer）
    • 中断/子程序时很重要

4.3 取指周期微操作（必须会写）

要会解释每一步在干什么：

  • t0：把PC内容送到MAR
  • t1：从存储器取指令到MBR，同时PC加1
  • t2：把MBR中的指令送到IR

4.4 执行周期重点命令（必须会）

1）LOAD addr

2）STA addr

3）ADD addr

4）JUMP addr

4.5 中断周期（高频）

中断本质

执行当前程序时，CPU被要求暂停当前工作，先处理别的请求。

必背流程

  1. 当前指令执行完
  2. 保存断点（通常保存PC）
  3. 跳转到ISR（中断服务程序）
  4. 中断结束后返回原程序继续执行

中断周期微操作

理解重点：

  • PC先保存
  • 再跳到ISR入口
  • 返回时恢复原程序

4.6 间址周期（Indirect Cycle）

含义：

  • 指令里给的不是最终地址
  • 需要先到存储器取出真正有效地址

4.7 流水线（Pipelining）

两级流水

  • 取指
  • 执行

四级流水

常见拆分：

  • IF：Instruction Fetch
  • ID：Instruction Decode
  • OF：Operand Fetch
  • EX：Execute

必背结论

  • 流水线的目标：提高吞吐率
  • 不一定能让单条指令更快
  • 会考：
    • 总时间计算
    • 加速比 Sp
    • 效率 E

4.8 指令格式与地址数（高频）

地址数分类

  • 1地址指令
  • 2地址指令
  • 3地址指令

典型比较

表达式：

  • 但指令长度更长、译码更复杂

六、第4章复习提纲：控制单元

6.1 必背核心概念

1）控制单元功能

  • 指令译码
  • 产生执行所需控制信号

2）微指令 / 微程序 / 例程

  • 微指令（microinstruction）：一个时钟周期内的微操作控制字
  • 微程序（microprogram）：微指令的集合
  • 例程（routine）：实现某个功能的一组微指令

6.2 控制单元结构（必须会背）

  • IR：提供操作码
  • Instruction Decoder：译码器
  • CAR：控制地址寄存器
  • Control Memory：控制存储器
  • CBR：控制缓冲寄存器
  • SBR：子程序寄存器
  • Sequencing Module：顺序控制模块

6.3 微指令格式（高频）

一般包括：

  • 运算字段（operation fields）
  • 条件字段（CD）
  • 分支字段（BR）
  • 地址字段（ADF）

你必须知道

条件字段 CD

  • U：无条件分支
  • I：若 I=1，则进入间址周期
  • S：AC符号位为1则分支
  • Z：AC为0则分支

分支字段 BR

  • JUMP
  • CALL
  • RET
  • MAP

6.4 映射（Mapping）

把机器指令的操作码映射成执行该指令的微程序入口地址。

典型考法：

  • 已知操作码
  • 问执行周期例程入口地址是多少

6.5 顺序控制（重点）

决定下一条微指令地址如何产生。

关键规则

  • BR = 00 / 01：
    • 若条件不满足 → 下一顺序地址
    • 若条件满足 → 跳转到 ADF
  • BR = 10：
    • 从 SBR 返回（RET）
  • BR = 11：
    • 由映射结果决定入口（MAP）

6.6 垂直微程序设计 vs 水平微程序设计

垂直型（Vertical）

优点

  • 微指令更短
  • 控制存储器容量更小

缺点

  • 需要译码器
  • 会增加延迟

水平型（Horizontal）

优点

  • 不需要译码
  • 速度快
  • 硬件相对简单

缺点

  • 微指令更长
  • 控制存储器更大

  • 高级语言经编译器变成机器语言，汇编语言经汇编器变成机器语言
  • 操作码位数决定最大操作种类
  • 地址位数决定最大可寻址空间

第2章

  • CPU = ALU + 寄存器组 + 控制单元
  • 取指周期：
  • 常见执行周期：
    • LOAD
    • STA
    • ADD
    • JUMP
  • 中断时先保存PC，再转ISR
  • 流水线提高吞吐率，不一定提高单条指令速度

第3章

  • 13.625 = 1101.101₂
  • 负数常用补码表示
  • 负数补码 = 取反 + 1
  • 8位补码范围：-128 ~ +127
  • 13 × 11 = 143 = 10001111₂
  • 减法可转为加补码
  • 溢出：正+正得负，负+负得正
  • 常见逻辑运算：AND / OR / XOR / NOT
  • 常见移位：逻辑左移、逻辑右移、算术右移

第4章

  • 控制单元负责译码和发控制信号
  • 微指令字段：运算字段、CD、BR、ADF
  • 映射：操作码 → 微程序入口地址
  • BR 常见：JUMP / CALL / RET / MAP
  • 垂直微程序：省空间但慢
  • 水平微程序：快但占空间
